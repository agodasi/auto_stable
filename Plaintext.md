# プロジェクト概要
Stable Diffusion Forge (API) をリモート操作し、画像生成のキュー（順番）を管理・自動化するデスクトップアプリケーションを開発してください。
最終的にPyInstaller等で `.exe` 化して配布・運用することを前提としたコード設計にしてください。

# 実行環境と技術スタック
- クライアント: Windows 11
- サーバー: 同一LAN内にあるUbuntu PC (SD Forgeを `--api` および `--listen` で起動中)
- 使用言語: Python 3.x
- GUIフレームワーク: PyQt6 または CustomTkinter (モダンなフラットデザイン、非同期処理対応)
- 通信: `requests` または `aiohttp`

# データ保存形式
- `config.json` : IP、保存先パス、テーマ、ベースパラメータの設定
- `presets.json` : 「シチュエーション」「キャラクター」のテキストプリセット
- `queue_state.json` : アプリ終了時のキュー状態（次回起動時のレジューム用）

# 必須機能要件

## 1. 設定画面（歯車アイコンから別ウィンドウまたはモーダルで開く）
- **システム設定:** API URL、保存先ディレクトリ、テーマ（ホワイト/ブラック）の切り替え。
- **ベース生成パラメータ:** Checkpoint（モデル）の取得・選択、ネガティブプロンプト、Steps、CFG Scale、Width / Height。
- **プリセット管理:** 登録済みの「シチュエーション」および「キャラクター」プリセットの一覧表示、テキストの編集、項目の削除が行える管理画面。

## 2. メイン画面：グローバル設定
全キュー共通で適用されるパラメータ。
- Batch count (数値入力)
- FreeU Integrated 設定 (Enableトグル、B1, B2, S1, S2 の数値入力)

## 3. メイン画面：キュー（待ち行列）システム
- **キューの追加（2種類のボタンを最下部に配置）:**
  1. `+ キューを追加` (ウィザード形式):
     - ステップ1: 「シチュエーション」をプルダウンから選択（「スキップ（なし）」も選択可能）し「次へ」。
     - ステップ2: 「キャラクター」をプルダウンから選択（「スキップ（なし）」も選択可能）し「完了」。
     - 結合: `[シチュエーションのテキスト]\n[キャラクターのテキスト]` の形式で新しいキューとして追加。
  2. `+ 空のキューを追加`: テキストが空のキューを即座に追加。
- **キューカードの挙動:**
  - 各カードは複数行入力可能で、自由にテキストを直接編集できる。
  - 「削除（ゴミ箱）」ボタン付き。
  - **編集ロック:** 生成処理が開始され「1番」で実行中になったカードのテキストボックスは編集不可（ReadOnly）になる。

## 4. メイン画面：プレビューとプリセット逆保存
- **プレビュー＆プログレス:** 画像プレビュー領域と、`/sdapi/v1/progress` をポーリングするプログレスバー（0〜100%）。
- **プリセット逆保存機能:** プレビュー付近に「現在のプロンプトをプリセット保存」ボタンを配置。
  - クリックすると、メインウィンドウの8割程度のサイズのモーダルダイアログが開く。
  - モーダル下部: 現在の画像のプロンプトをReadOnlyで表示（テキスト選択・コピー可能）。
  - モーダル上部（ステップ1）: 「シチュエーション」の入力欄。下部からコピーしたテキストを貼り付け、新規名または「既存プリセットへの上書き」を選んで「次へ（またはスキップ）」。
  - モーダル上部（ステップ2）: 「キャラクター」の入力欄。同様に新規か上書きを選んで「保存」。

## 5. 実行コントロールと生成ロジック
- 「生成開始」ボタンを押すと一番上のキューから処理開始。保存先ディレクトリの存在チェック（マウント確認）を行い、なければエラーダイアログを出して中断。
- 非同期処理により、生成中も後続キューの追加・編集が可能。
- 生成中はボタンが「キャンセル」に変わり、クリックで `/sdapi/v1/interrupt` を送信。
- 1番の生成が完了（またはキャンセル）するとカードが消え、2番が繰り上がる。
- APIへのPOST時、Seed値は常に `-1`、メタデータ（PNG info）の書き込みは不要。
- デコードした画像を `YYYYMMDD_HHMMSS_{連番}.png` で保存。

# 開発の進め方
1. `requirements.txt` の提示とアーキテクチャの解説。
2. GUIのベースレイアウト（設定、グローバル設定、キューエリア、プレビュー）のモックアップ。
3. キュー追加ウィザードと、プリセット逆保存の8割サイズモーダルのUI実装。
4. `config.json` / `queue_state.json` を用いた状態保存・復元。
5. API通信（FreeUパラメータ含む）、ディレクトリチェック、非同期生成、キャンセル機能の実装。
6. 最終コードの提示とPyInstallerでのビルドコマンド案内。
